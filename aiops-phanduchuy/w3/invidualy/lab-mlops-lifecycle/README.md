To run the MLOps lifecycle pipeline from start to finish: first bring up the core infrastructure stack using `bash scripts/start_stack.sh`, train and register the baseline anomaly detection model using `uv run python duckq1u/pipeline.py --data data/baseline.csv`, start the FastAPI production serving server with `uv run python duckq1u/serve.py`, and finally execute the retraining and monitoring pipeline using `uv run python duckq1u/retrain.py --reference data/baseline.csv --current data/drifted.csv --holdout data/holdout.csv --post-deploy-eval data/post_deploy_eval.csv` which runs combined data and performance drift check, trains a robust sliding-window candidate v2 model, verifies baseline holdout precision, prompts for deployment approval, performs a zero-downtime blue-green reload, and runs post-deployment auto-rollback evaluation on live predictions.
